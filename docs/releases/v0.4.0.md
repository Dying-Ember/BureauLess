# BureauLess v0.4.0 — Audit Comparison Contract v3

This release makes cross-Agent comparisons capable of measuring adaptive
context behavior without weakening the controlled experiment boundary.

## Highlights

- Keeps every dispatch candidate in the `routing_mode` decision dimension and
  records Agent/model/provider as fixed bindings.
- Adds `bureauless_benchmark_identity_v3`, separating initial context from
  realized context requests, resolutions, and estimated added tokens.
- Adds strict `fixed-context` and explicit `adaptive-context` paired comparison
  modes.
- Classifies declared versus observed execution changes as single treatment,
  multi treatment, treatment mismatch, or descriptive only.
- Retains task and initial-context payloads so their hashes can be recomputed.
- Accepts verified archive manifests as comparison inputs and labels raw
  session inputs as unverified.
- Adds an optional HMAC-SHA256 route-configuration fingerprint without storing
  provider URLs.

## Compatibility

- Historical benchmark identity v1 and v2 sessions remain readable.
- New capability-contribution artifacts require benchmark identity v3.
- No local Agent configuration, provider URL, or credential value is stored.
- No npm or PyPI artifact is published by this release.

## Evidence boundary

- Adaptive context is measured as process evidence, not treated as controlled
  initial context.
- Capability comparisons still state `causal_claim: not_established`.
- Raw session inputs remain explicitly distinct from hash-verified archives.
- A route fingerprint is unavailable unless the operator supplies a stable
  local `BUREAULESS_ROUTE_FINGERPRINT_KEY`.

## Verification

- Local backend regression: 294 tests passed.
- The release tag is created only after the GitHub `backend` and `workbench`
  checks pass on the release commit.

## 中文摘要

v0.4.0 将 initial context 与运行过程中实际获取的 context 分开，使 BureauLess
既能做严格 fixed-context 对比，也能在 adaptive-context 模式下测量 context 请求次数
与新增 token。决策候选现在处于同一 routing-mode 维度；comparison 会检查 treatment
声明、输入完整性与 route 指纹，同时继续拒绝伪因果结论。
