# BureauLess v0.3.0 — Audit Evidence Contract v2

This release strengthens BureauLess as a cross-Agent measurement harness. It
does not add another Agent; it makes existing runs more honest to compare and
more precise about what the Harness did or did not observe.

## Highlights

- Adds `bureauless_benchmark_identity_v2` with separate task, delivered-context,
  execution, workspace-baseline, and independent-acceptance identities.
- Adds controlled fields, execution treatment differences, and explicit
  uncontrolled confounders to paired capability comparisons.
- Records dispatch candidate sets, rejection reasons, budget/risk basis, and
  which choices were fixed by the operator.
- Moves telemetry and comparison eligibility to Agent×Provider route evidence.
- Replaces scalar session-route verification with dimension-specific launch,
  request, mutation, telemetry, model, cost, permission, and event evidence.
- Replaces coarse side-effect observation with scoped coverage and explicit
  blind spots for workspace, process, network, credential, and payment effects.
- Applies the same persisted dispatch-binding validation to every real Agent.
- Rejects URLs, paths, whitespace, and common credential shapes in opaque route
  and cohort labels.

## Compatibility

- Historical benchmark identity v1 and side-effect coverage records remain
  readable.
- New capability-contribution artifacts require benchmark identity v2 because
  v1 does not control delivered context or expose execution treatment changes.
- No local Agent configuration file is modified, and no provider URL or secret
  value is stored in benchmark identity.

## Evidence Boundary

- Dispatch decisions are Harness-owned evidence. Agent-internal decisions are
  not inferred from final prose.
- `partial` side-effect coverage means only the named scope was observed; its
  `blind_spots` remain unknown.
- Paired comparisons continue to state `causal_claim: not_established`.
- Adapter implementations remain co-located in `runtime/sessions.py`; physical
  module splitting is deferred until ownership or change frequency warrants it.

## Verification

- Backend: 292 tests passed.
- Workbench: Web and desktop builds passed; 40 Playwright smoke tests passed.
- Release validation does not call a real Agent or provider and requires no
  provider credential.
