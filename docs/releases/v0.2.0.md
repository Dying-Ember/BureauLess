# BureauLess v0.2.0 — Cross-agent Audit Harness

BureauLess now provides one evidence contract around heterogeneous coding-agent
runtimes. It stays outside their model and tool loops while making route
selection, workspace effects, verification, telemetry provenance, and
cross-run comparison inspectable.

## Highlights

- Registers Codex CLI, Claude Code, Gemini CLI, OpenCode, and Pi under one
  Agent×Provider route model.
- Separates Agent, Provider route, endpoint family, wire API, model identity,
  credentials, and adapter support instead of treating a provider name as a
  compatibility guarantee.
- Runs supported routes through isolated child workspaces with one-shot
  configuration; local Agent configuration files are not modified.
- Retains native output and workspace state as evidence, then derives
  provenance-labelled facts without replacing the originals.
- Adds Harness-owned independent verification, append-only route observations,
  decision points, side-effect coverage, benchmark identity, and
  identity-matched paired comparisons.
- Upgrades Claude Code telemetry to its native `stream-json` lifecycle so tool
  use, tool results, permission denials, usage, cache, and cost remain visible.

Run `bureauless agent matrix --evidence` for current machine-readable state.
The table above is a release summary, not a substitute for route evidence.

## Evidence Boundary

- Native logs remain evidence; normalized facts preserve provenance.
- Workspace diff, not a tool event or final prose, determines final file state.
- Requested and reported model identities remain separate.
- Missing usage or currency cost remains unavailable.
- Secrets are supplied to child processes by environment-variable name and are
  not stored in the registry.
- Capability comparisons report measurable deltas and state
  `causal_claim: not_established`.

## Known Boundaries

- OpenCode's Anthropic Messages runtime shape is observed, but its BureauLess
  adapter is not implemented.
- OpenCode and Pi expose Responses-shaped runtime configuration, but the tested
  endpoint instance was unavailable; those adapters remain unimplemented.
- Provider-attributed telemetry is available on the maintained Codex Responses
  route. Other adapters retain their native Agent-reported provenance.
- Dry-run records `review_status: not_run` and unavailable cost provenance; it
  proves artifact materialization, not Agent or endpoint execution.
- Reported monetary cost is telemetry, not verified payment or charge evidence.
- BureauLess is not an Agent runtime, provider gateway, fallback mesh, secret
  manager, or universal billing oracle.

## 中文摘要

v0.2.0 将 BureauLess 的主场景明确为跨 Agent 外部审计：用同一套 registry、route、
证据和比较契约包住五个 coding Agent，同时不接管 Agent 内部的模型循环与工具执行。
本版本提供隔离执行、原生证据保留、独立验证、追加式 route observation、benchmark
identity、side-effect coverage 和有 provenance 的 paired comparison；未验证或未实现的
route 继续明确标记为不可用，不通过 Provider 品牌推断兼容性。
