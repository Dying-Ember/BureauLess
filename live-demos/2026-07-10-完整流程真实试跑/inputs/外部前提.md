# 外部前提

本文件只记录任务发布人明确提供、允许系统依赖的外部前提。

它**不是**控制平面答案，也**不是** routing 决策。

## 已知条件

- provider 类型：OpenAI Compatible
- provider base URL：`https://jojocode.com/v1`
- provider key：通过环境变量提供，不在文档中记录具体值
- 结果与记录语言：中文

## 允许系统使用的事实

- 可以使用现有仓库中的 demo fixture 作为起点。
- 可以在本次 run 自己的 `workspace/` 内创建或修改文件。
- 可以在系统允许的情况下提出新的 workflow proposal、assignment 或 mutation proposal。
- 可以在系统允许的情况下请求 review、verification、acceptance 所需的显式产物链。

## 未被外部预先指定的事项

以下事项不应被视为任务发布人已经替系统决定：

- 使用哪一种 workflow mode
- 是否沿用现有 demo workflow
- 是否需要新增 verifier / reviewer / committer 之外的新 role
- 是否需要 mutation
- 具体 assignment 如何拆分
- 派生 worker 使用什么模型

## 本次运行的审计立场

任务发布人关心的是：

- 系统是否如实上报新增 role / agent 的申请
- orchestrator 是否按 harness 规则行事
- worker 是否按 assignment 权限行事
- harness 是否真的阻止越权，而不只是事后记录

任务发布人**不**替系统内部角色做控制平面判断。
